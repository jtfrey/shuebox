sendDataPart:

- (SBError*) sendDataPart:(SBData*)data;
Discussion

Invoke this method after message headers have been sent in order to add a MIME part containing the given binary data. The MIME part has a content type of "application/octet-stream" associated with it.

The content-disposition will be set to "inline".

sendDataPart:mimeType:

- (SBError*) sendDataPart:(SBData*)data mimeType:(SBString*)mimeType;
Discussion

Invoke this method after message headers have been sent in order to add a MIME part containing the given binary data. An explicit mimeType can be provided (e.g. "application/zip"); if mimeType is nil, the content type will be set to "application/octet-stream".

The content-disposition will be set to "inline".

sendDataPart:mimeType:filename:

- (SBError*) sendDataPart:(SBData*)data mimeType:(SBString*)mimeType 
        filename:(SBString*)filename;
Discussion

Invoke this method after message headers have been sent in order to add a MIME part containing the given binary data. An explicit mimeType can be provided (e.g. "application/zip"); if mimeType is nil, the content type will be set to "application/octet-stream".

The content-disposition will be set to "attachment" as long as filename is a valid string. Otherwise (or if filename is nil) the "inline" disposition is used.

sendFromAddress:

- (SBError*) sendFromAddress:(SBString*)fromAddress;
Discussion

Invoke this method one time only after a new SMTP session has been setup to send the "MAIL FROM" command with the given email address.

sendMessage:withProperties:

- (SBError*) sendMessage:(SBString*)message withProperties:(SBDictionary*)messageProps;
Discussion

Convenience method which sends the given message body using the receiver's default SMTP host/port. Message meta data are drawn from the messageProps dictionary, including recipients, sender adddress, subject, etc.

sendMessage:withSubject:

- (SBError*) sendMessage:(SBString*)message withSubject:(SBString*)subject;
Discussion

Convenience method which sends the given message body and subject to the receiver's default recipient using the default SMTP host/port and the default sender address.

sendMessageHeaders:

- (SBError*) sendMessageHeaders:(SBDictionary*)messageProps;
Discussion

Given an SBDictionary containing message meta data keyed by the string constants defined at the end of this header file, send the DATA command followed by all applicable SMTP headers.

This method should be invoked one time only after at least one of the recipient sending methods have been invoked.

sendMessageWithComposer:properties:

- (SBError*) sendMessageWithComposer:(id)composer properties:(SBDictionary*)messageProps;
Discussion

Akin to sendMessage:withProperties:, this method acts as a message-building driver. It opens a session, sends all sender and recipient information and meta data as drawn from the messageProps dictionary. If successful to that point, it invokes the addMessagePartsToMailer: method on the composer object.

The composer object must conform to the SBMailerComposer protocol and is responsible for adding content parts to the message. It should keep track of errors as it proceeds and return an SBError object if it needs to signal that the message could not be composed properly.

sendMessageWithComposerFunction:context:properties:

- (SBError*) sendMessageWithComposerFunction:(SBMailerMultipartComposerFunction)composer 
        context:(void*)composerContext properties:(SBDictionary*)messageProps;
Discussion

Akin to sendMessage:withProperties:, this method acts as a message-building driver. It opens a session, sends all sender and recipient information and meta data as drawn from the messageProps dictionary. If successful to that point, it calls the composer function.

The composer function is responsible for adding content parts to the message. It should keep track of errors as it proceeds and return an SBError object if it needs to signal that the message could not be composed properly.

sendRecipient:

- (SBError*) sendRecipient:(SBString*)toAddress;
Discussion

Invoke this method after the sendFromAddress: method has been invoked to send a "RCPT TO" command with the provided email address. Can be invoked multiple times.

sendRecipients:

- (SBError*) sendRecipients:(SBArray*)toAddresses;
Discussion

Invoke this method after the sendFromAddress: method has been invoked to send a "RCPT TO" command for each of the provided email addresses in the toAddresses array. Can be invoked multiple times.

sendTextPart:

- (SBError*) sendTextPart:(SBString*)text;
Discussion

Invoke this method after message headers have been sent in order to add a MIME part containing the given text. The MIME part has a content type of "text/plain" associated with it.
