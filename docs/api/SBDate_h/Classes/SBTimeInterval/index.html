SBTimeInterval

Superclass: SBObject
Declared In: SBDate.h

Discussion

Instances of SBTimeInterval wrap a time interval. Time intervals may be created as second, day, or month counts.

daysInTimeInterval

- (SBInteger) daysInTimeInterval;
Discussion

Returns the number of days (as an int) in the receiver's time interval. An interval that does not fall on a day boundary will be rounded down.

monthsInTimeInterval

- (SBInteger) monthsInTimeInterval;
Discussion

Returns the number of months (as an int) in the receiver's time interval. An interval that does not fall on a month boundary will be rounded down.

secondsInTimeInterval

- (double) secondsInTimeInterval;
Discussion

Returns the number of seconds (as a double) in the receiver's time interval.

timeIntervalWithDays:seconds:

+ (id) timeIntervalWithDays:(SBInteger)days seconds:(double)seconds;
Discussion

Returns an initialized, autoreleased instance of SBTimeInterval which wraps a number of days. The count is integer-based so this is whole days (24 hours) only; an interval that does not fall on a day boundary will always be rounded down.

timeIntervalWithMonths:days:seconds:

+ (id) timeIntervalWithMonths:(SBInteger)months days:(SBInteger)days 
        seconds:(double)seconds;
Discussion

Returns an initialized, autoreleased instance of SBTimeInterval which wraps a number of months. The count is integer-based so this is whole months (30 days) only; an interval that does not fall on a month boundary will always be rounded down.

timeIntervalWithSeconds:

+ (id) timeIntervalWithSeconds:(double)seconds;
Discussion

Returns an initialized, autoreleased instance of SBTimeInterval which wraps a number of seconds.

totalSecondsInTimeInterval

- (double) totalSecondsInTimeInterval;
Discussion

Returns the total number of seconds in the time interval, with day and month counts converted to seconds and coallesced into the total.
