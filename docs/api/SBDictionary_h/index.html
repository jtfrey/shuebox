SBDictionary.h

Discussion

SBDictionary represents the public interface to an entire cluster of classes devoted to the task of managing keyed collections of objects. Where an SBArray uses a fixed range of integer indices to identify constituent values, an SBDictionary allows any SBFoundation object to act as the identifier for the values in the container.

Implementation Details

A little inside info, the actual class cluster under SBDictionary looks like this:
  • SBDictionary
    • SBNullDictionary
    • SBConcreteDictionary
      • SBSinglePairConcreteDictionary
      • SBSmallConcreteDictionary
      • SBMediumConcreteDictionary
      • SBLargeConcreteDictionary

    • SBMutableDictionary

      • SBConcreteMutableDictionary

There are two abstract subclasses of SBDictionary. SBNullDictionary represents an immutable dictionary which contains no key-value pairs.

The concrete implementations of SBDictionary (SBSinglePairConcreteDictionary, et al.) differ only in their capacity and how they store their constituent objects. The single-pair, small, and medium types all include a static C data structure in their instance variable list, so no additional allocations are necessary beyond creating the array object itself. These three classes have capacities of 1, 8, and 24 key-value pairs, respectively. The SBLargeConcreteDictionary will handle any object capacity and allocates its storage space from the heap.
