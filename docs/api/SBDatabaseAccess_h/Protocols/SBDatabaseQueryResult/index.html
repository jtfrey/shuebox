SBDatabaseQueryResult

Declared In: SBDatabaseAccess.h

Discussion

Groups methods which any database class' query result class(es) should implement. The methods are general enough that they should be applicable to any database engine.

arrayForFieldNum:

- (SBArray*) arrayForFieldNum:(SBUInteger)fieldNum;
Discussion

Convenience method which should invoke arrayForFieldNum:createMutable: with the createMutable flag being NO.

arrayForFieldNum:createMutable:

- (SBArray*) arrayForFieldNum:(SBUInteger)fieldNum 
        createMutable:(BOOL)createMutable;
Discussion

For a data-returning query (e.g. SELECT), extract the value of a given field (column) for all rows of the returned query result. The values are added to the resulting array in the order in which they occur in the result set.

If createMutable is YES, then the returned array object will be an SBMutableArray.

arrayForRow:

- (SBArray*) arrayForRow:(SBUInteger)row;
Discussion

Convenience method which should invoke arrayForRow:createMutable: with the createMutable flag being NO.

arrayForRow:createMutable:

- (SBArray*) arrayForRow:(SBUInteger)row createMutable:(BOOL)createMutable;
Discussion

For a data-returning query (e.g. SELECT), process all columns in the given row as they would be processed in the objectForRow:fieldNum: method and return the resulting objects in an array ordered according to field number.

If createMutable is YES, then the returned array object will be an SBMutableArray.

classForFieldNum:

- (Class) classForFieldNum:(SBUInteger)fieldNum;
Discussion

For a data-returning query (e.g. SELECT), inspect the data type associated with the given column and return the ObjC class which would be used to wrap the value. For example, a column with a floating point type would return the SBNumber class while a BLOB would return the SBData class.

Returns SBData as the default.

copyValueAtRow:fieldNum:toBuffer:length:

- (SBUInteger) copyValueAtRow:(SBUInteger)row fieldNum:(SBUInteger)fieldNum 
        toBuffer:(void*)buffer length:(SBUInteger)length;
Discussion

For a data-returning query (e.g. SELECT), copy the first "length" bytes of the value in the given row and column to "buffer". If "length" is larger than the actual byte size of the value, the remainder of "buffer" is untouched.

Returns the actual number of bytes copied to "buffer".

dictionaryForRow:

Discussion

Convenience method which should invoke dictionaryForRow:createMutable: with the createMutable flag being NO.

dictionaryForRow:createMutable:

- (SBDictionary*) dictionaryForRow:(SBUInteger)row 
        createMutable:(BOOL)createMutable;
Discussion

For a data-returning query (e.g. SELECT), process all columns in the given row as they would be processed in the objectForRow:fieldNum: method and return the resulting objects in a dictionary, keyed by field names.

If createMutable is YES, then the returned array object will be an SBMutableDictionary.

fieldNameWithNumber:

- (SBString*) fieldNameWithNumber:(SBUInteger)fieldNum;
Discussion

For a data-returning query (e.g. SELECT), returns the name associated with column number fieldNum; columns use a zero-based numbering scheme.

fieldNumberWithName:

- (SBUInteger) fieldNumberWithName:(SBString*)fieldName;
Discussion

For a data-returning query (e.g. SELECT), returns the column number which has the given fieldName associated with it. Note that the treatment of case may vary between database systems.

getSizeOfValue:andPointer:atRow:fieldNum:

        atRow:(SBUInteger)row fieldNum:(SBUInteger)fieldNum;
Discussion

Combination of the sizeOfValue:atRow:fieldNum: and getValuePointer:atRow:fieldNum: methods to get both size and pointer in one pass.

getValuePointer:atRow:fieldNum:

- (BOOL) getValuePointer:(void**)valuePtr atRow:(SBUInteger)row 
        fieldNum:(SBUInteger)fieldNum;
Discussion

For a data-returning query (e.g. SELECT), sets valuePtr to a pointer to the returned value in the given row and column. Note that this is often a pointer to memory internal to the specific database's query result data structure, so the pointer should not be considered generally thread-safe or long-term viable. If you need the data to stick around, make a copy.

If the method was able to set valuePtr, YES is returned. Otherwise, NO is returned.

Discussion

For a data-returning query (e.g. SELECT), returns YES if the value in the given row and column is NULL.

objectForRow:fieldNum:

- (id) objectForRow:(SBUInteger)row fieldNum:(SBUInteger)fieldNum;
Discussion

For a data-returning query (e.g. SELECT), inspect the data type associated with the given column and return an object of the appropriate ObjC class that wraps the value in the given row and column. For example, a TEXT or VARCHAR field would return an SBString object.

queryWasSuccessful

- (BOOL) queryWasSuccessful;
Discussion

Returns YES if the receiver represents a successful SQL query.

sizeOfValue:atRow:fieldNum:

- (BOOL) sizeOfValue:(SBUInteger*)byteSize atRow:(SBUInteger)row 
        fieldNum:(SBUInteger)fieldNum;
Discussion

For a data-returning query (e.g. SELECT), sets byteSize to the number of bytes used to represent the returned value in the given row and column.

If the method was able to set byteSize, YES is returned. Otherwise, NO is returned.
