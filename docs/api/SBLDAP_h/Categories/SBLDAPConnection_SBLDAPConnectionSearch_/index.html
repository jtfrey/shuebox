SBLDAPConnection(SBLDAPConnectionSearch)

Extends Class: SBLDAPConnection
Declared In: SBLDAP.h

Discussion

Category which groups LDAP search methods.

Possible return values for all methods are:

  • SBError: returned if any error occurred during the search
  • SBLDAPSearchResult: the search produced a single directory entity
  • SBArray: the search produced multiple directory entities

searchWithFilter:

- (id) searchWithFilter:(SBString*)ldapFilter;
Discussion

Search the receiver's LDAP directory tree for directory entities matching the provided LDAP filter string. The default base DN for the receiver and subtree scope are implied. Attribute values will also be retrieved.

searchWithFilter:baseDN:

- (id) searchWithFilter:(SBString*)ldapFilter baseDN:(SBString*)baseDN;
Discussion

Search the receiver's LDAP directory tree (from the given base DN) for directory entities matching the provided LDAP filter string. Subtree scope is implied. Attribute values will also be retrieved.

If baseDN is nil, the default base DN for the receiver is used.

searchWithFilter:baseDN:scope:

- (id) searchWithFilter:(SBString*)ldapFilter baseDN:(SBString*)baseDN 
        scope:(int)scope;
Discussion

Search the receiver's LDAP directory tree (from the given base DN) for directory entities matching the provided LDAP filter string. The provided search scope is observed. Attribute values will also be retrieved.

If baseDN is nil, the default base DN for the receiver is used.

searchWithFilter:baseDN:scope:ignoreAttributes:

- (id) searchWithFilter:(SBString*)ldapFilter baseDN:(SBString*)baseDN 
        scope:(int)scope ignoreAttributes:(BOOL)ignoreAttributes;
Discussion

Search the receiver's LDAP directory tree (from the given base DN) for directory entities matching the provided LDAP filter string. The provided search scope is observed. Attribute values will only be retrieved if ignoreAttributes is NO.

If baseDN is nil, the default base DN for the receiver is used.
