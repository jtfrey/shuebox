SBXMLNode

Superclass: SBObject
Declared In: SBXMLNode.h

Discussion

The root of a class cluster that is used to represent XML document pieces. An SBXMLNode has a distinct sub-type ("kind") that identifies what sub-class models it, etc.

All SBXMLNode objects have a subordinate relationship with a parent SBXMLNode, with the exception of SBXMLDocumentNode objects. An SBXMLNode also has an index that identifies its position among any sibling nodes.

The value of an SBXMLNode is subject to its kind. For example, a node of type kSBXMLNodeKindAttribute stores its attribute value in this property as an SBString.

SBXMLNode instances should not be allocated directly. Always use the class factory methods to ensure that the appropriate SBXMLNode subclass is used.
