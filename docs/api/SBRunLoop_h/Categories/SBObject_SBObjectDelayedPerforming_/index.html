SBObject(SBObjectDelayedPerforming)

Extends Class: SBObject
Declared In: SBRunLoop.h

Discussion

Extensions to SBObject that allow for delayed invocation.

cancelPreviousPerformRequestsWithTarget:

+ (void) cancelPreviousPerformRequestsWithTarget:(id)aTarget;
Discussion

Remove all queue invocations of aTarget from the current runloop.

cancelPreviousPerformRequestsWithTarget:selector:object:

+ (void) cancelPreviousPerformRequestsWithTarget:(id)aTarget 
        selector:(SEL)aSelector object:(id)anArgument;
Discussion

Remove all queue invocations of aSelector on aTarget (with single argument anArgument) from the current runloop.

performSelector:withObject:afterDelay:

- (void) performSelector:(SEL)aSelector withObject:(id)anArgument 
        afterDelay:(SBTimeInterval*)delay;
Discussion

Augmented version of performSelector:withObject: that queues the invocation in the current runloop to be performed after the given time interval.

performSelector:withObject:afterDelay:inModes:

- (void) performSelector:(SEL)aSelector withObject:(id)anArgument 
        afterDelay:(SBTimeInterval*)delay inModes:(SBArray*)modes;
Discussion

Augmented version of performSelector:withObject: that queues the invocation in the current runloop to be performed after the given time interval and only when the current runloop is in a mode present in the modes array.
