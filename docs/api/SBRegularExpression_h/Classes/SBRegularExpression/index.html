SBRegularExpression

Superclass: SBObject
Declared In: SBRegularExpression.h

Discussion

Class which wraps an ICU Unicode regular expression. Unicode regular expressions behave pretty much just like POSIX extended regular expressions, with some added character set support junk. See:

http://icu-project.org/apiref/icu4c/uregex_8h.html

Instances of SBRegularExpression can be initialized using either an SBString object or a UTF8-compliant C string. Some special flags can be optionally passed to the instance; the flags affect case sensitivity, newline treatment, word boundaries, etc. See:

http://icu-project.org/apiref/icu4c/uregex_8h.html#874989dfec4cbeb6baf4d1a51cb529ae

Once an instance has been initialized, it can be used to process one or more SBString objects. The string to be processed -- the "subject string" -- is set and (optionally, if ICU version 4 is being used) a sub-range of the full string can be selected. Repeated calls to "findNextMatch" will incrementally locate the next sub-string matching the regular expression and return YES; a return value of NO indicates no matches remain. Methods exist to determine the extent of the current match (full or partial, relative to the subject string) and to retrieve the matched character range and/or any grouped character ranges within the match.

findNextMatch

- (BOOL) findNextMatch;
Discussion

Repeatedly returns YES if at least some portion of the remaining un-matched portion of the subject string (or the set matching range on the subject string) matches the receiver's regular expression. Contrast this method against the isFullMatch and isPartialMatch methods, which only ever operate on the subject string in its entirety.

If YES, then the stringForMatch and stringForMatchingGroup: methods can be used to retrieve matching sub-strings.

isFullMatch

- (BOOL) isFullMatch;
Discussion

Returns YES if the entirety of the subject string (or the set matching range on the subject string) matches the receiver's regular expression. If YES, then the stringForMatch and stringForMatchingGroup: methods can be used to retrieve matching sub-strings.

isPartialMatch

- (BOOL) isPartialMatch;
Discussion

Returns YES if at least some portion of the subject string (or the set matching range on the subject string) matches the receiver's regular expression. If YES, then the stringForMatch and stringForMatchingGroup: methods can be used to retrieve matching sub-strings.

matchingGroupCount

- (SBUInteger) matchingGroupCount;
Discussion

Returns the number of grouped character ranges that appear in the receiver's regular expression.

matchingRange

- (SBRange) matchingRange;
Discussion

Returns the range of characters within the subject string to which the regular expression matching operations should be restricted.

rangeOfMatch

- (SBRange) rangeOfMatch;
Discussion

After a successful matching method has been invoked, the method returns the range of characters in the subject string which were matched by the receiver's regular expression.

rangeOfMatchingGroup:

- (SBRange) rangeOfMatchingGroup:(SBUInteger)groupNum;
Discussion

After a successful matching method has been invoked, the method returns the range of characters in the subject string which were matched by one of the grouped character ranges in the receiver's regular expression. Groups are indexed from one up to and including the value returned by the matchingGroupCount method.

rangesOfMatchingGroups:

- (void) rangesOfMatchingGroups:(SBRange*)groupRanges;
Discussion

After a successful matching method has been invoked, the method copies the array of parenthetical sub-matching ranges to groupRanges. Note that groupRanges must be at least as large as the number of parenthetical sub-groups plus one.

resetMatching

- (BOOL) resetMatching;
Discussion

Reset internal state on the receiver's regular expression; subsequent matching methods will search from the beginning of the subject string once again. Returns YES if the internal state was reset properly.

setMatchingRange:

- (BOOL) setMatchingRange:(SBRange)range;
Discussion

Modified the range of characters within the subject string to which the regular expression matching operations should be restricted.

setSubjectString:

- (void) setSubjectString:(SBString*)subject;
Discussion

Resets the receiver's regular expression matching state and retains the given subject string to use for subsequent matching.

Note that the subject string's characters are NOT copied; if you modify the subject string itself while attempting to process matches the behavior is undefined.

stringForMatch

- (SBString*) stringForMatch;
Discussion

After a successful matching method has been invoked, this method returns a string containing the full range of characters in the subject string which were matched by the receiver's regular expression.

stringForMatchingGroup:

- (SBString*) stringForMatchingGroup:(SBUInteger)groupNum;
Discussion

After a successful matching method has been invoked, this method returns a string containing one of the grouped character ranges in the subject string which were matched by the receiver's regular expression. Groups are indexed from one up to and including the value returned by the matchingGroupCount method.

subjectString

- (SBString*) subjectString;
Discussion

Returns the SBString currently associated with the receiver for regular expressiong matching.
