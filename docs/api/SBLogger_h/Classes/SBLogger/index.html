SBLogger

Superclass: SBObject
Declared In: SBLogger.h

Discussion

Instances of SBLogger provide a unified interface by which low-level programmatic messaging can be performed. The standard example is a file containing a trace of operations performed by a program -- a log file. Each SBLogger has a minimum priority assigned to it: any messages with lower priority are discarded and not logged. For example, setting the minimum priority to kSBLoggerPriorityError will avoid any debug, notice, and warning messages being added.

Conceptually, this class performs the same duties as UNIX syslog. Indeed, instances of this class can be created that send messages to the syslog facility. The class is more generic, though, so the same API can be used to target syslog or a simple on-disk file.
