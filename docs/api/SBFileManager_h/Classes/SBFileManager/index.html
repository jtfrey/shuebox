SBFileManager

Superclass: SBObject
Declared In: SBFileManager.h

Discussion

The SBFileManager class is designed to streamline interactions with file attributes. A single shared instance is provided, so consumer code should never need to directly allocate an instance of this class.

contentsAtPath:

- (SBData*) contentsAtPath:(SBString*)path;
Discussion

Attempts to open the file at path and read its content into memory. If successful, an SBData object containing the file's content is returned. Otherwise, nil is returned.

createDirectoryAtPath:

- (BOOL) createDirectoryAtPath:(SBString*)path;
Discussion

If path is not an extant UNIX file path, create a new directory at path with default permissions and ownership. Returns YES if successful.

createDirectoryAtPath:mode:

- (BOOL) createDirectoryAtPath:(SBString*)path mode:(mode_t)mode;
Discussion

If path is not an extant UNIX file path, create a new directory at path with default ownership and permissions specified by mode. Returns YES if successful.

createFileAtPath:contents:

- (BOOL) createFileAtPath:(SBString*)path contents:(SBData*)data;
Discussion

Attempts to open the file at path for writing (creating it if not present) and writes the contents of the data object to the file. Returns YES if successful.

directoryExistsAtPath:

- (BOOL) directoryExistsAtPath:(SBString*)path;
Discussion

Returns YES if the receiver represents an extant UNIX file path and the entity is a directory (not a socket, file, symlink, etc.).

enumeratorAtPath:

- (SBDirectoryEnumerator*) enumeratorAtPath:(SBString*)path;
Discussion

Returns an SBDirectoryEnumerator that walks the contents of the directory at path.

fileExistsAtPath:

- (BOOL) fileExistsAtPath:(SBString*)path;
Discussion

Returns YES if the receiver represents an extant UNIX file path and the entity is a file (not a socket, directory, symlink, etc.).

isExecutableFileAtPath:

- (BOOL) isExecutableFileAtPath:(SBString*)path;
Discussion

Returns YES if the receiver represents a UNIX file path that is deemed "executable" by the program (via the access() function).

isReadableFileAtPath:

- (BOOL) isReadableFileAtPath:(SBString*)path;
Discussion

Returns YES if the receiver represents a UNIX file path that is deemed "readable" by the program (via the access() function).

isWritableFileAtPath:

- (BOOL) isWritableFileAtPath:(SBString*)path;
Discussion

Returns YES if the receiver represents a UNIX file path that is deemed "writable" by the program (via the access() function).

movePath:toPath:

- (BOOL) movePath:(SBString*)src toPath:(SBString*)dest;
Discussion

Attempts to rename the entity at src to dest. Note that this method will fail and return NO if the two paths reside on different filesystems!

openPath:asCFileStreamWithMode:

- (FILE*) openPath:(SBString*)path asCFileStreamWithMode:(const char*)mode;
Discussion

Attempts to open the file at the UNIX file path represented by the receiver. Returns NULL if the file could not be opened.

openPath:withFlags:mode:

- (int) openPath:(SBString*)path withFlags:(int)openFlags 
        mode:(mode_t)mode;
Discussion

Attempts to open the file at the UNIX file path represented by the receiver. Returns NULL if the file could not be opened.

ownerGIdAtPath:

Discussion

ownerUIdAtPath:

- (uid_t) ownerUIdAtPath:(SBString*)path;
Discussion

Returns the UNIX user id number of the user-owner of the entity at path.

pathExists:

- (BOOL) pathExists:(SBString*)path;
Discussion

Returns YES if the receiver represents an extant UNIX file path.

posixPermissionsAtPath:

- (mode_t) posixPermissionsAtPath:(SBString*)path;
Discussion

If path is an extant UNIX file path, returns the permissions mask of the entity.

removeFileAtPath:

- (BOOL) removeFileAtPath:(SBString*)path;
Discussion

Attempts to delete the entity at the given UNIX file path. Returns YES if the entity is not a directory and the current user had permission to delete the entity.

removeItemAtPath:

- (BOOL) removeItemAtPath:(SBString*)path;
Discussion

Attempts to delete the entity at the given UNIX file path. For non-directory entities, behaves just like removeFileAtPath:. For a directory, the directory and all of its content (sub-directories and files) are removed. Returns YES if successful.

setOwnerUId:andGId:atPath:

- (BOOL) setOwnerUId:(uid_t)userId andGId:(gid_t)groupId 
        atPath:(SBString*)path;
Discussion

If path is an extant UNIX file path, attempts to set the user-owner of the entity to userId and the group-owner to groupId. Returns YES if successful.

setOwnerUId:andGId:posixMode:atPath:

- (BOOL) setOwnerUId:(uid_t)userId andGId:(gid_t)groupId 
        posixMode:(mode_t)mode atPath:(SBString*)path;
Discussion

If path is an extant UNIX file path, attempts to set the user-owner of the entity to userId and the group-owner to groupId. Also sets the permissions mask to mode. Returns YES if successful.

Computationally more efficient than separate calls to setOwnerUId:andGId:atPath: and setPosixPermissions:atPath:.

setOwnerUId:atPath:

- (BOOL) setOwnerUId:(uid_t)userId atPath:(SBString*)path;
Discussion

If path is an extant UNIX file path, attempts to set the user-owner of the entity to userId. Returns YES if successful.

setPosixPermissions:atPath:

- (BOOL) setPosixPermissions:(mode_t)mode atPath:(SBString*)path;
Discussion

If path is an extant UNIX file path, attempts to set the entity's permissions mask to mode. Returns YES if successful.
