SBFileHandle

Superclass: SBObject
Declared In: SBFileHandle.h

Discussion

SBFileHandle is an abstract base class that represents input and output channels, typically embodied by files. A file handle can be readable, writable, or both. The class uses a single read/write pointer that (in general) is positionable at arbitrary offsets.

The majority of the SBFileHandle methods will raise an SBException on error conditions; all such exceptions are named according to the string constant SBFileHandleOperationException.

Behind the scenes there is a hierarchy of private concrete subclasses that implement the different entities to which an SBFileHandle could be associated.

availableData

- (SBData*) availableData;
Discussion

If the receiver is readable and any data is avalable for reading, do so and return it wrapped in an SBData object.

closeFile

- (void) closeFile;
Discussion

Close the file associated with the receiver. If the receiver is writable, all modified data and attributes will be flushed to the storage media.

isReadable

- (BOOL) isReadable;
Discussion

Returns boolean YES if data can be read from the receiver.

isWritable

- (BOOL) isWritable;
Discussion

Returns boolean YES if data can be written to the receiver.

offsetInFile

- (SBFileOffset) offsetInFile;
Discussion

Returns the receiver's current read/write file pointer.

readDataOfLength:

- (SBData*) readDataOfLength:(SBUInteger)length;
Discussion

If the receiver is readable, read at most length bytes (or to end-of-file) and return the data read as an SBData object.

readDataToEndOfFile

- (SBData*) readDataToEndOfFile;
Discussion

If the receiver is readable, read until end-of-file is reached and return the data read as an SBData object.

seekToEndOfFile

- (SBFileOffset) seekToEndOfFile;
Discussion

Attempt to reposition the receiver's file pointer at the end-of-file and, if successful, return the offset of that end-of-file.

seekToFileOffset:

- (void) seekToFileOffset:(SBFileOffset)offset;
Discussion

Attempt to reposition the receiver's file pointer to the given offset relative to the start of the file.

synchronizeFile

- (void) synchronizeFile;
Discussion

If the receiver is writable, force all modified data and attributes to be flushed to the storage media behind the file (see fsync()).

truncateFileAtOffset:

- (void) truncateFileAtOffset:(SBFileOffset)offset;
Discussion

If the receiver is writable, set it to be offset bytes in length.

writeData:

- (void) writeData:(SBData*)data;
Discussion

If the receiver is writable, write the contents of the data object at the current offset.
