initWithBytes:objCType:

- (id) initWithBytes:(const void*)bytes objCType:(const char*)type;
Discussion

Initialize the receiver to wrap the specified data type. The type is analyzed to determine the size of the type (in bytes); a size-appropriate buffer is allocated and filled with that much data from "bytes".

valueWithBytes:objCType:

+ (SBValue*) valueWithBytes:(const void*)value objCType:(const char*)type;
Discussion

Returns a newly-initialized, autoreleased instance which wraps the given data and type.

The type can be computed using the @encode() compiler directive. For example, to create an SBValue which wraps a 2D point: 

typedef struct {
  float   x,y;
} point2d_t;

point2d_t   aPoint = { 1.0 , -1.0 };
SBValue*    asValue = [SBValue valueWithBytes:&aPoint objCType:@encode(point2d_t)];

valueWithNonretainedObject:

+ (SBValue*) valueWithNonretainedObject:(id)anObject;
Discussion

Convenience method which returns a newly-initialized, autoreleased instance which wraps the given object without increasing that object's reference count. Of course, one needs to be extremely careful with this, since the contents of the SBValue may or may not become invalid if anObject is released!

valueWithPointer:

+ (SBValue*) valueWithPointer:(const void*)pointer;
Discussion

Convenience method which returns a newly-initialized, autoreleased instance which wraps the given pointer. Of course, one needs to be extremely careful with this: if the wrapped pointer were on the stack then eventually the references pointer would go out of scope. Likewise, if the wrapped pointer is a malloc()'ed buffer, that buffer could be free()'d and the SBValue would be left with an invalid pointer!
