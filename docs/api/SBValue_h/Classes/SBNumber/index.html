boolValue

- (BOOL) boolValue;
Discussion

Returns the value of the receiver as a boolean -- zero equates to NO, non-zero equates to YES.

compare:

- (SBComparisonResult) compare:(SBNumber*)otherNumber;
Discussion

Compare the value of the receiver to the value of otherNumber. The nature of the comparison is dictated by the underlying atomic type of the receiver. In other words, if the receiver wrapped a BOOL, the value of the receiver is compared to the boolValue of otherNumber.

doubleValue

- (double) doubleValue;
Discussion

Returns the value of the receiver as a double-precision floating-point value.

int64Value

- (int64_t) int64Value;
Discussion

Returns the value of the receiver as a signed 64-bit integer. Values outside the range of an int64_t are truncated to the min/max value.

integerValue

- (SBInteger) integerValue;
Discussion

Returns the value of the receiver as an SBInteger. Values outisde the range of an SBInteger are truncated to the min/max value.

intValue

- (int) intValue;
Discussion

Returns the value of the receiver as a signed integer. Values outside the range of an int are truncated to the min/max value.

isEqualToNumber:

- (BOOL) isEqualToNumber:(SBNumber*)number;
Discussion

Returns YES if [self compare:number] is SBOrderedSame -- so equality is based on the same conditions outlined for the compare: method.
